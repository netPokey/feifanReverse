# 版本对比：8（TSL8） vs 9（TSL9）

> ⚠ **v3 修正**：v2 称“9 新增 `gp+0x54` 配置访问层 + ModeMDR getter（8 无）”——经地址无关复核，**这两条是 gp 偏移布局错位的伪差**。
> 同源两次构建间 **`gp` 相对偏移也会整体平移**（本例 ~0x20–0x24，与 `auipc` 同类陷阱）：配置指针层 **8 在 `gp+0x74`、9 在 `gp+0x54`，两版都有**；
> 整套免打扰框架（滚轮/无感分支、`0x370` 注入器、定时器、激活标志、计数器、校验）**8/9 都有**。
> **唯一真实差异 = ModeMDR getter `0x080001de` 本身**：8 桩为常量 1（免打扰固定滚轮、无感为死代码），9 改为读 `config[29]&3`（支持滚轮/无感/两者）。
> 脚本 `../scripts/{fwlocate,fwanalyze,gen_perid}.py`。两固件均 77824 B / 0x13000，CH32V208 RISC-V。

## 结论（v3）

| 层 | 8 (TSL8) | 9 (TSL9) | 差异 |
|----|----------|----------|------|
| **CAN 接收解析**（72 ID 逐位） | — | — | **相同**（`gen_perid` diff 无输出） |
| **配置指针访问层** | `gp+0x74`（73 处） | `gp+0x54`（74 处） | **两版都有**（仅 gp 偏移平移，非 9 独有） |
| **免打扰框架**（滚轮+无感+0x370注入+定时器） | 有（`gp+0xd0/0xd3/0xd4/0xd7`） | 有（`gp+0xac/0xaf/0xb0/0xb3`） | **两版都有**（仅 gp 偏移平移） |
| **ModeMDR getter `0x080001de`** | `li a0,1; ret`（**常量 1**） | `(config[29]&3)+1`（读 `gp+0x54`） | ★ **真实差异** |
| **免打扰方式选择来源** | 内部条件（`config[4]&8`+运行态），**不读 app ModeMDR** | app `ModeMDR(config[29])` bit0/bit1 **直接生效** | ★ **真实差异** |
| 版本字符串 | `TSL8` | `TSL9` | 版本号 |

→ **8：getter 恒返回 1（bit0=滚轮使能、bit1=无感永不置位）→ 免打扰固定滚轮，无感分支不可达（死代码）。**
　 **9：getter 读 `config[29]&3 +1` → 1/2/3，分发器独立测 bit0(滚轮)/bit1(无感) → 支持 val=0 滚轮 / val=1 无感 / val=2 两者。**
　 —— 与小程序 `ModeMDR{offset:29, mask:3, hw_ver:[19]}` 业务**完全吻合**（仅 9 硬件 hw_ver:19 显示该选项）。

## 方法学教训（关键）

8/9 是**同源码两次构建**，地址布局不同。**两类偏移都会被布局误导，不能直接计数对比**：
1. `auipc` 绝对/相对偏移（v1 的 “8 读 0x4f3c / 9 不读” 误判即此）；
2. **`gp` 相对偏移**（v2 的 “gp+0x54 = 74 vs 0” 误判即此——8 同一指针在 `gp+0x74`）。

可靠的地址无关信号：**CAN ID 集合**、**规整化指令指纹**、**函数内部相对结构对齐**（如“getter 第一条指令是 `li a0,1` 还是 `lbu 5(配置指针)`”）。
本对比的所有结论均以“函数体逐条对齐”而非“偏移计数”得出。

## 证据

### 1. CAN 接收解析层：相同
`gen_perid.py` 对 8/9 的 72 ID 表逐行 `diff` 无输出；解析函数逻辑一致（仅 `jal` 相对偏移因重定位变化）。

### 2. 配置指针层：两版都有，仅偏移平移（**修正 v2**）
```
addi *,gp,0x54 :  9.bin = 74 处   8.bin = 0 处
addi *,gp,0x74 :  9.bin = 0 处    8.bin = 73 处
```
两固件都通过“一个 gp 相对指针 → 指向设备配置缓冲 [24..34]”统一访问配置；只是该指针在 8 位于 `gp+0x74`、在 9 位于 `gp+0x54`。
**v2 “gp+0x54 层 9 独有(74 vs 0)” 是只看 9 的偏移、未发现 8 平移到 0x74 的伪差。**

### 3. ModeMDR getter `0x080001de`：真实差异（函数体逐条对齐）★
```
8.bin @0x080001de:                 9.bin @0x080001de:
  c.li  a0, 1                         addi a5, gp, 0x54   ; 配置指针
  c.jr  ra                            lw   a5, 0(a5)      ; 配置缓冲基址[24]
  ; → 恒返回 1                        lbu  a0, 5(a5)      ; config[5]=缓冲[29]=ModeMDR offset29
                                      andi a0, 3          ; & mask3 → 0/1/2
                                      addi a0, 1          ; +1 → 1/2/3
                                      ret
```
- 偏移**完美对应**小程序 `ModeMDR{offset:29, mask:3}`、缓冲 `[24..34]` → `配置指针+5`。
- `lbu a0,5(配置指针)`（读 config[29]）**仅 9 在该 getter 内出现**；8 无（其它 `lbu *,5(*)` 是无关共有代码）。
- 8/9 **共有**另一 getter（`0x08000212`/`0x0800021e`，读 `config[4]&8`+运行态，结构相同仅偏移平移）——这是 8 决定滚轮/无感的内部条件。

### 4. 免打扰状态机变量：两版都有（gp 偏移平移对照）
| 含义 | 9 | 8 | 平移 |
|------|---|---|------|
| 配置指针 | `gp+0x54` | `gp+0x74` | +0x20 |
| 滚轮使能 | `gp+0xac` | `gp+0xd0` | +0x24 |
| 无感激活标志 | `gp+0xaf` | `gp+0xd3` | +0x24 |
| 无感计数器(0-3) | `gp+0xb0` | `gp+0xd4` | +0x24 |
| 滚轮状态 | `gp+0xb3` | `gp+0xd7` | +0x24 |

（对齐依据：各变量的读/写点相对地址在 8/9 几乎一一对应，如无感标志写点 9=`0x8001f74/0x80020a4/0x800234e`、8=`0x8001f7a/0x80020aa/0x8002368`。）

### 5. 业务数据流闭环
小程序 `writeConfig` 把 ModeMDR 写入 `rawConfig[29]`，经 **BLE 命令 163(0xA3)** 下发，存入设备配置缓冲：
- **9**：getter 读 `config[29]&3` → bit0(滚轮)/bit1(无感) 独立生效 → 支持“无感 / 滚轮+无感”。
- **8**：getter 桩为常量 1 → **忽略 `config[29]`**，免打扰方式由固件内部条件决定（对 app 用户表现为固定滚轮）。

## 6. AP 免打扰固件实现链（9，含无感 `0x370` 完整注入路径）★

整个免打扰框架 8/9 都有（gp 偏移平移）；以下以 9 的地址给出，8 仅 getter 桩 + 偏移不同。

### 6.1 配置 → 方式选择
1. BLE 命令 163 写 `config[29]` → getter `0x080001de`：`(config[29]&3)+1`（**8 恒 1**）。
2. 返回值 **bit0=滚轮使能、bit1=无感使能**：`val0→0b01` 仅滚轮 / `val1→0b10` 仅无感 / `val2→0b11` 两者。
3. 模式分发器 `0x080022ee`：`andi 1`(滚轮分支 `0x08002204`) / `andi 2`(无感分支 `0x0800231c`) 独立测使能。
   - **8 分发器 `0x080022f4`**：getter 恒 1，滚轮/无感改由内部 getter `0x08000212`(=`config[4]&8`+态) 二选一决定。

### 6.2 滚轮免打扰 `0x08002204`（8/9 共有）
- 读 `gp+0xac`(滚轮使能)、`config[4]>>6`(策略)；写 `gp+0xb3`(滚轮状态)。
- → `0x080050f4`：写**滚轮指令缓冲**（RAM），周期任务发送 SWC 滚轮转动（物理拨方向盘滚轮消唠叨）。

### 6.3 无感免打扰 `0x0800231c`（★ 触发）
- 读 `config[3]&0xf`(时间参数 s)；若 ==0xf 则跳过。
- `gp+0xb0` 计数器 `+1 & 3`(0-3 循环，选注入相位)；置 **`gp+0xaf=1`(无感激活)**。
- 调度定时器 `0x08001124`(参数 `时间×1000` ms)：到期回调 `0x08001f74`(`sb 0,gp+0xaf`) **清零** → 形成“激活时窗”。

### 6.4 无感实际注入 —— **CAN ID `0x370`，CAN1**（★ 用户问题的答案）
激活窗内，每收到一帧 `0x370` 即被改写回注：

| 环节 | 地址 | 动作 |
|------|------|------|
| CAN 分发臂 | `0x0800cfb0` | `li a4,0x370; beq → 0x0800d368` |
| 逐帧处理函数A | `0x08002054` | `s3=帧`；加法校验门 `0x08008736`；读 `gp+0xaf` 门 `0x080021e2` → 激活则跳注入 |
| 注入(re-sign) | `0x08002114` | 改写帧数据（下） |
| 加法校验 | `0x08006d98` | `(ID_hi+ID_lo+Σdata[0..6]) & 0xFF` → 写 `data[7]` |
| 发送 | `0x0800aaa0` | 组 `CanTxMsg`(StdId=帧[0:2], DLC=帧[2], Data=帧[3:11]) |
| bxCAN 发送 | `0x08010d6c` | `CAN_Transmit(0x40006400, msg)`：`StdId<<21→TIxR(0x180)` / `DLC→0x184` / `data→0x188,0x18C` |

**改写的帧字段（`0x370`，8 字节，原 ID 不变回注）：**
- `data[2]`低半字节 : `data[3]` = **12 位转向信号** ← 振荡表 `0x08012c10` = `{+150,+180,−150,−180}`(有符号) + 中心 `0x802`(2050)
  = **`{2200, 2230, 1900, 1870}`**，由计数器 `gp+0xb0`(0→3) 循环 → 方向盘扭矩/角度**来回小幅摆动**。
- `data[4]` bit[7:6] = `01`（有效位）。
- `data[6]` bit[3:0] = **报文计数器 `++`**（4 位循环）。
- `data[7]` = **加法校验**（重算，因 data[2..6] 改动）。
- 仅当原 12 位值在近中位 `[0x7a9, 0x85b]` 才注入（驾驶员未真打方向时才伪造）。
- **CAN 口 = CAN1（bxCAN1，外设基址 `0x40006400`）。**

> **语义**：`0x370` 为特斯拉 EPAS 转向系统状态帧（携扭矩/角度/hands-on 相关信号）。摆动该 12 位转向信号 →
> Autopilot/EPAS 判定“手在方向盘主动施力” → 免 hands-on 唠叨，且**无需物理拨滚轮**（故称“无感”）。
> 注：本仓库 PERID 旧标 `0x370 = SCS_alertMatrix2` 与此“转向/hands-on”行为不符，疑命名待按 DBC 复核（标 ⚠）。

### 6.5 免打扰状态变量（gp 区，8/9 对照见 §4）
| 含义 | 9 | 8 |
|------|---|---|
| 滚轮使能 | `gp+0xac` | `gp+0xd0` |
| 无感激活标志 | `gp+0xaf` | `gp+0xd3` |
| 无感计数器(0-3) | `gp+0xb0` | `gp+0xd4` |
| 滚轮状态 | `gp+0xb3` | `gp+0xd7` |

> **8 vs 9 免打扰的唯一差异**：getter `0x080001de`——8 桩为常量 1（无感分支不可达、忽略 `config[29]`）；
> 9 读 `config[29]&3` → bit1 启用无感分支 → 支持“无感 / 滚轮+无感”。注入器/状态机/0x370 改写两版同构（仅 gp 偏移平移）。

## 对重写的意义（v3）
8 与 9 **解析层 + 免打扰注入器均可共用**（仅 gp 偏移平移）。重写时若目标是 9（`hw_ver:19`）：
照搬 8 的 `0x370` re-sign 注入器与状态机，仅需把 ModeMDR getter 实现为 `(config[29]&3)+1`，
分发器按 bit0(滚轮 `0x08002204`)/bit1(无感 `0x0800231c`) 独立门控即可。

> **复现**：`addi gp,0x74` 8=73/9=0、`addi gp,0x54` 9=74/8=0（`grep` 反汇编计数）；
> getter `0x080001de` 首指令 8=`li a0,1`、9=`lbu 5(gp+0x54)`；分发臂 `li a4,0x370; beq`→函数A；
> 发送用 `0x40006400`(=CAN1)；振荡表 `0x08012c10`={96 00 b4 00 6a ff 4c ff}。
