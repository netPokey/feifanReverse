# 版本对比：8（TSL8） vs 9（TSL9）

> ⚠ **v2 修正**：初版结论“CAN 功能完全相同”**只对了一半**。CAN **接收解析层**确实相同，但
> **9 新增了 `gp+0x54` 配置访问层 + AP 免打扰（ModeMDR）读取，8 没有** —— 这是真实功能差异。
>
> **方法教训**：8/9 地址布局不同（同源两次构建），**基于 `auipc` 绝对/相对偏移的对比会被布局误导**
> （初版即因此误判“完全相同”）。版本对比必须用**地址无关信号**：`gp` 相对偏移、规整化指令指纹、CAN ID 集合。
> 脚本 `../scripts/{fwlocate,fwanalyze,gen_perid}.py`。两固件均 77824 B / 0x13000，CH32V208 RISC-V。

## 结论（修正版）

| 层 | 8 (TSL8) | 9 (TSL9) | 差异 |
|----|----------|----------|------|
| **CAN 接收解析**（72 ID 逐位） | — | — | **相同**（`gen_perid` diff 无输出） |
| **配置访问层 `gp+0x54`** | **0 处** | **74 处** | ★ **9 独有** |
| **AP 免打扰 ModeMDR getter** | **无** | `0x1ea`（gp+0x54→`config[5]&3 +1`） | ★ **9 独有** |
| 全固件规整化指令相似度 | — | — | 95.36%（9 多 44 条指令、31 个独有逻辑块） |
| 版本字符串 | `TSL8` | `TSL9` | 版本号 |

→ **9 支持 AP 免打扰方式选择（val=0/1/2：滚轮 / 无感 / 滚轮+无感），8 只能 val=0（滚轮）**
　 —— 与小程序 `ModeMDR{offset:29, mask:3, hw_ver:[19]}` 业务**完全吻合**。

## 证据

### 1. CAN 接收解析层：相同（初版结论成立的部分）
`gen_perid.py` 对 8/9 的 72 ID 表（命名/类型/逐位读取）逐行 `diff` 无输出；解析函数逻辑一致
（仅 `jal` 相对偏移因重定位变化）。**接收→解析→状态链路两版一致**。

### 2. 配置访问层 `gp+0x54`：9 独有（地址无关，74 vs 0）★
`gp` 相对偏移**不受地址布局影响**，是可靠信号：
```
9.bin:  addi *,gp,0x54（配置缓冲指针）= 74 处，遍布全固件 16 个代码区
8.bin:  0 处
```
9 通过 `gp+0x54`（指向设备配置缓冲 [24..34] 起点的指针）统一访问各配置项；8 无此访问层。

### 3. ModeMDR（AP 免打扰方式）getter：9 独有 ★
9 的 `0x080001de`：
```
addi a5, gp, 0x54   ; a5 = &配置指针
lw   a5, 0(a5)      ; a5 = 设备配置缓冲基址（[24]）
lbu  a0, 5(a5)      ; config[5] = 缓冲[24+5] = [29] = ModeMDR offset29
andi a0, 3          ; & mask3 → 免打扰方式 0/1/2
addi a0, 1          ; +1（转 1/2/3 供内部分支）
ret
```
偏移**完美对应**小程序 `ModeMDR{offset:29, mask:3}`、缓冲 `[24..34]` → `配置指针+5`。
- 8/9 **共有**一个 getter（8@`0x483e` / 9@`0x48a4`）读 `config[1]`（=[25]，另一配置项），逻辑一致；
- 9 **额外新增** `0x1ea` 这个读 `config[5]=[29]=ModeMDR` 的 getter（经 `gp+0x54`）——**8 没有**。

### 4. 业务数据流闭环
小程序 `writeConfig` 把 ModeMDR 写入 `rawConfig[29]`，经 **BLE 命令 163(0xA3)** 下发，存入设备配置缓冲：
- **9**：`gp+0x54` 读 `config[29]&3` 得 val（0/1/2）→ 走对应免打扰分支 → 支持“无感 / 滚轮+无感”。
- **8**：无此 getter，免打扰固定为默认（val=0 滚轮）。

## 对重写的意义（修正）
8 与 9 **CAN 接收解析层可共用**，但 **9 多了配置访问层（`gp+0x54`）与 ModeMDR 等配置驱动的行为分支**。
重写时：解析层照搬 8；若目标是 9（`hw_ver:19`），需补 `gp+0x54` 配置访问 + ModeMDR(`config[29]&3`) 的免打扰多模式分支。

> **复现**：`gp+0x54` 用量差异可由 `grep 'addi.*gp, 0x54'` 反汇编计数（8=0 / 9=74）；
> ModeMDR getter 见 9@`0x080001de`（读 `config[5]&3+1`），8 无对应。

## 5. 补充：9 的 `gp+0x54` 配置访问层完整覆盖（对照 settingsOptions）

9 通过 `gp+0x54` 指针统一访问整个设备配置区（缓冲 [24..34] = config[0..10]），各字节读取频次与含义：

| config偏移 | 缓冲offset | 9读取 | 配置项（`settingsOptions.js`） |
|-----------|-----------|-------|------------------------------|
| config[0] | 24 | 6 | 能量回收制动 |
| config[1] | 25 | 5 | AP 关闭自动雨刮 |
| config[2] | 26 | 2 | 提示音音量(8bit) |
| config[3] | 27 | 8 | LED 灯 / 滚轮策略 |
| config[4] | 28 | 14 | 模块总开关 / AP 恢复方式 / 滚轮随机 |
| **config[5]** | **29** | 2 | **ModeMDR 免打扰(bit0-1,mask3)★ + 长拉Ns(其他bit)** |
| config[6] | 30 | 8 | 延时恢复AP / 远光闪3次 / 开门·倒车双闪灯 |
| config[7] | 31 | 11 | 一键AP / 随心远光 |
| config[8] | 32 | 5 | (扩展配置) |
| config[9] | 33 | 4 | 动态刹车灯 / 空调烘干 |
| config[10] | 34 | 2 | 安全带提醒 |

> - 9 引入**统一配置访问抽象层**（`gp+0x54`），覆盖全部 11 字节配置（AP辅助/灯光/免打扰/电控等）；8 无此层（分散 `auipc` 直读）。
> - **ModeMDR（`config[29]&3`）取值 getter 为 9 独有**（`0x080001de`），8 无对应 → 免打扰多模式仅 9 支持（与 `hw_ver:[19]` 一致）。
> - 注意 `offset29` 字节复用：bit0-1=ModeMDR、其他 bit=“长拉Ns”等；故 8 的 `config[29]==5` 整字节检查 ≠ ModeMDR 取值。
> - offset30/31 含大量 AP 辅助+灯光增强（延时恢复AP/一键AP/随心远光/双闪灯），9 读取频繁；各项 8/9 支持差异可按 `hw_ver` 逐项续查。

## 6. ModeMDR 免打扰固件实现链（9，沿 ModeMDR 线深入）

完整追踪 AP 免打扰在固件 9 的实现（8 无 getter，免打扰固定默认滚轮）：

### 配置 → 使能位
1. BLE 命令 163 写 `config[29]` → getter `0x080001de`：`(config[29]&3)+1`。
2. 返回值 **bit0=滚轮使能、bit1=无感使能**：
   - `val0→1`(0b01)=仅滚轮 / `val1→2`(0b10)=仅无感 / `val2→3`(0b11)=滚轮+无感。
3. 3 个调用者（`0x08002082 / 0x08002302 / 0x08002fa2`）用 `andi 1`(滚轮) / `andi 2`(无感) 测使能分支。
   核心是 `0x08002302`：分别走滚轮、无感两条注入路径。

### 滚轮免打扰 `0x08002204`（8/9 共有）
- 读 `gp+0xac`(滚轮使能)、`config[4]>>6`(AP 恢复/滚轮策略)；写 `gp+0xb3`(滚轮状态)。
- → `0x080050f4`：写**滚轮指令缓冲**（`auipc 0x1ffff` RAM 区 4 字节：标志/方向 `a0`/量 `-a2`/参数 `a1`），供周期任务发送滚轮转动。

### 无感免打扰 `0x0800231c`（★ 9 独有路径）
- 读 `config[3]&0xf`(时间参数)；`gp+0xb0` 计数器(0-3 循环)；设 `gp+0xaf=1`(无感激活标志)。
- 触发无感注入（方向盘扭矩/触摸帧），让 Autopilot 误判“手在方向盘”，从而免打扰但不转动滚轮。

### 免打扰状态变量（`gp` 区，地址无关）
| gp 偏移 | 含义 |
|--------|------|
| `gp+0xac` | 滚轮使能 |
| `gp+0xaf` | 无感激活标志 |
| `gp+0xb0` | 无感计数器(0-3) |
| `gp+0xb3` | 滚轮状态 |

> **8 vs 9 免打扰的唯一差异**：8 无 ModeMDR getter，免打扰固定滚轮（val=0）；
> 9 经 `config[29]&3` 取 ModeMDR，**bit1 启用无感分支（9 独有）** → 支持“无感 / 滚轮+无感”。
> CAN 接收解析层两版完全相同，差异纯在此**配置驱动的免打扰状态机**。
