# 微信小程序 wxapkg 解包产物 → 可调试工程 还原笔记

> 目标：把"解包后的散文件"还原成微信开发者工具能**导入、调试、发布**、且**人类可读**的标准小程序工程。
> 成品目录：`weapp-rebuild/`

---

## 0. TL;DR（最短路径）

```bash
# 1. 准备纯净输入（只需 3 个核心文件）
mkdir -p /tmp/wxinput
cp app-config.json app-service.js page-frame.html /tmp/wxinput/

# 2. 按 wxapkg 格式重新打包（脚本见第 3 节）
node /tmp/repack.js          # 产出 /tmp/wxpkg/__APP__.wxapkg

# 3. 反编译
npx -y wedecode@latest /tmp/wxpkg -o /tmp/wxoutput --clear

# 4. 补真实资源 + 落地 + 清理 + 修组件引用（见第 5~7 节）
```

## 1. 先搞清楚：手上的到底是什么

解包目录里有一堆文件，但**它们不是可读源码**，而是新版 wxapkg 的"appservice / webview 分离格式"运行时产物。

| 文件 | 真实身份 | 是否需要 |
|---|---|---|
| `app-config.json` | 全局 + 各页面配置（合并态） | ✅ 反编译输入 |
| `app-service.js` | 所有 JS 逻辑，`define("path", fn)` 模块格式 | ✅ 反编译输入 |
| `page-frame.html` | 所有 WXML（`$gwx` 生成器函数）+ WXSS（`setCssToHead` 编译态）+ JSON | ✅ 反编译输入 |
| `chunk_*.appservice.js` / `chunk_*.webview.js` | 运行时 `$gwx` 分片 | ❌ 冗余中间产物 |
| `*.app.js`（common/appservice/webview） | 运行时框架 | ❌ 冗余 |
| `pages/**/.html`、`components/**/.html` | **加载器占位**，不是模板 | ❌ 冗余 |
| `assets/**/*.png` | **真实资源图片** | ✅ 反编译器拿不到，需手动补回 |

### 关键判断命令

```bash
# 看目录结构 & 文件类型分布
find . -maxdepth 1 -type d
find pages -type f | sed 's/.*\.//' | sort | uniq -c

# 确认页面 html 只是加载器（里面是 $gwx_XC_N + __wxAppCode__ 脚本，没有模板）
cat pages/index/index.html

# 确认 JS 是 define() 模块格式
grep -o 'define("[^"]*"' app-service.js | head -40

# 确认 page-frame 里 WXML/WXSS/JSON 的存放形式
python3 -c "
data=open('page-frame.html',encoding='utf-8',errors='replace').read()
for key in [\"__wxAppCode__['components/app-modal/app-modal.wxml']\",
            \"__wxAppCode__['components/app-modal/app-modal.json']\"]:
    i=data.find(key); print(repr(data[i:i+200]))
"

# 确认有没有分包（本项目无）
grep -o '"subPackages"\|"subpackages"' app-config.json
```

**结论**：
- JSON → `__wxAppCode__['x.json'] = {...}`，直接可还原
- WXSS → `setCssToHead([...])` 编译态，需反解为 CSS
- WXML → `$gwx_XC_N` 生成器函数，**最难，必须用专门反编译器**

## 2. 为什么不手写反解，而用 wedecode

`$gwx` 是腾讯把 WXML 编译成的指令式生成函数，手写逆向极易出错。成熟方案是 `wedecode`（专门处理 appservice / page-frame 分离格式）。

```bash
# 工具自检（确认可联网 npx 拉取）
npx -y wedecode@latest --help
```

**坑点**：`wedecode` 只吃 `.wxapkg` 包文件，不吃已解包的散目录。

```
❌ 文件夹下不存在 .wxapkg 包 /tmp/wxinput
```

解决办法：把散文件**按 wxapkg 格式重新打包**回去（解包产物本就是包内容）。

## 3. 重打包脚本 `/tmp/repack.js`

wxapkg 文件格式：

```
header: 0xBE | uint32 info1 | uint32 indexLen | uint32 bodyLen | 0xED   (共 14 字节)
index:  uint32 fileCount | [ uint32 nameLen | name | uint32 offset | uint32 size ]...
body:   各文件数据顺序拼接
```

```javascript
const fs = require('fs');
const path = require('path');

const inputDir = '/tmp/wxinput';
const files = ['app-config.json', 'app-service.js', 'page-frame.html'];

const entries = files.map(f => ({
  name: '/' + f,
  data: fs.readFileSync(path.join(inputDir, f)),
}));

// index 段长度
let indexLen = 4;
for (const e of entries) indexLen += 4 + Buffer.byteLength(e.name) + 4 + 4;
let bodyLen = entries.reduce((s, e) => s + e.data.length, 0);

// header
const header = Buffer.alloc(14);
header.writeUInt8(0xBE, 0);
header.writeUInt32BE(0, 1);          // info1
header.writeUInt32BE(indexLen, 5);
header.writeUInt32BE(bodyLen, 9);
header.writeUInt8(0xED, 13);

// index + body
const indexBuf = Buffer.alloc(indexLen);
let p = 0;
indexBuf.writeUInt32BE(entries.length, p); p += 4;
const bodyStart = 14 + indexLen;
let dataOffset = bodyStart;
const bodyParts = [];
for (const e of entries) {
  const nameBuf = Buffer.from(e.name, 'utf8');
  indexBuf.writeUInt32BE(nameBuf.length, p); p += 4;
  nameBuf.copy(indexBuf, p); p += nameBuf.length;
  indexBuf.writeUInt32BE(dataOffset, p); p += 4;
  indexBuf.writeUInt32BE(e.data.length, p); p += 4;
  bodyParts.push(e.data);
  dataOffset += e.data.length;
}

const out = Buffer.concat([header, indexBuf, ...bodyParts]);
fs.mkdirSync('/tmp/wxpkg', { recursive: true });
fs.writeFileSync('/tmp/wxpkg/__APP__.wxapkg', out);
console.log('打包完成:', out.length, 'bytes | 文件数', entries.length);
```

```bash
node /tmp/repack.js
```

## 4. 反编译

```bash
npx -y wedecode@latest /tmp/wxpkg -o /tmp/wxoutput --clear
```

常用参数：
- `-o <dir>`  输出目录
- `--clear`   清空旧产物
- `--px`      用 px 解析 css（默认 rpx，**本项目保持 rpx**）
- `--unpack-only`  只解包不反编译

产出：`app.js / app.json / app.wxss`、`pages/**`、`components/**`、`services/**`、`utils/**`、`@babel/**`，WXML/WXSS/JS/JSON 四件套俱全。

## 5. 补真实资源（反编译器拿不到二进制）

反编译器只有 3 个核心文件，`assets/` 里的 161 个 PNG 必须从原始解包目录拷回。

# 原始目录有真实图片，输出目录只有占位 manifest.json

## 7. 修复组件引用瑕疵（关键）

反编译器对 `usingComponents` 的相对路径处理有缺陷：

- `../../components/xxx` —— 页面引用顶层组件，**正确**
- `components/app-modal/app-modal` —— 无 `../` 前缀，靠"在每个使用目录下复制一份组件副本"支撑（产生 25 份冗余副本），且 `custom-tab-bar/` 缺副本会**解析失败报错**

**修复策略**：统一改成绝对路径 `/components/...`（任何层级都能解析），删除全部嵌套副本。

```python
# 在输出目录运行
import json, glob, os, re, shutil

# 1) 删除嵌套副本目录（保留唯一顶层 components）
nested = [d for d in glob.glob('**/components', recursive=True) if d != 'components']
for d in sorted(nested, key=len, reverse=True):
    shutil.rmtree(d)

# 2) usingComponents 相对路径 -> 绝对路径
for jf in glob.glob('**/*.json', recursive=True):
    if os.path.basename(jf).startswith('._'): continue
    try: d = json.load(open(jf, encoding='utf-8'))
    except: continue
    uc = d.get('usingComponents')
    if not isinstance(uc, dict): continue
    mod = False
    for k, v in list(uc.items()):
        if not isinstance(v, str) or v.startswith('plugin://'): continue
        m = re.search(r'components/[^/]+/[^/]+$', v)
        if m and not v.startswith('/'):
            uc[k] = '/' + m.group(0); mod = True
    if mod:
        json.dump(d, open(jf, 'w', encoding='utf-8'), ensure_ascii=False, indent=2)
```

## 8. 最终质量校验

```bash
cd weapp-rebuild

# (a) 清理 macOS ._ 垃圾文件（cp -R 会再生成）
find . -type f -name '._*' -delete

# (b) JS 语法校验：应 0 错误
find . -name '*.js' -not -name '._*' | while read -r f; do node --check "$f" || echo "ERR $f"; done

# (c) 组件引用 0 缺失（绝对路径解析）
python3 - <<'PY'
import json,glob,os
miss=0
for jf in glob.glob('**/*.json',recursive=True):
    if os.path.basename(jf).startswith('._'): continue
    try: d=json.load(open(jf,encoding='utf-8'))
    except: continue
    for name,p in (d.get('usingComponents') or {}).items():
        if p.startswith('plugin://'): continue
        rp=p[1:] if p.startswith('/') else os.path.normpath(os.path.join(os.path.dirname(jf),p))
        if not (os.path.exists(rp+'.wxml') or os.path.exists(rp+'.js')):
            print('缺失',jf,name,p); miss+=1
print('缺失组件引用:',miss)
PY

# (d) WXSS 应是纯 CSS（无 setCssToHead / __wxAppCode__ 残留）
find . -name '*.wxss' -not -name '._*' -exec grep -l 'setCssToHead\|__wxAppCode__' {} \; | wc -l

  for ext in js json wxml wxss; do [ -f "$p.$ext" ] || echo "缺 $p.$ext"; done
done
```

## 9. 已知局限

- **JS 变量名是单字母**（`r`/`n`/`t`）：原包发布前已 minify，原始命名永久丢失，无法恢复。但模块结构、`require`/`module.exports`、业务逻辑完整，可读可调试。
- **WXML 是反编译重建**：语义等价，但格式/属性顺序可能与原始手写源码略有差异。
- **资源依赖原始解包目录**：PNG 等二进制不在反编译范围内，必须手动补。

## 10. 交付与使用

1. 微信开发者工具 →「导入项目」→ 选 `weapp-rebuild/`
2. 把 `project.config.json` 的 `"appid": "touristappid"` 换成自己的 AppID（否则只能预览、无法发布）
3. 确认运行正常后，原始解包目录的中间产物（`chunk_*.js`、`*.app.js`、`page-frame.html`、`app-service.js`、`app-config.json`、各 `.html` 占位）均可删除
