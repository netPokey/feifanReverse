# 控制注入 (re-sign) — 逆向发现记录

> 设备核心控车功能: 拦截特斯拉 CAN 帧 → 改写指定位 → 原 ID 回注。对照 9.bin。

## 两条控制注入路径 + AP 注入
1. **AP/免打扰 注入** (步① 已厘清):
   - 激活标志 `0x28003fb8`(共享)由 **免打扰调度器出口** `0x0800236e` 脉冲置位
     (pulse fn `0x080088ae`: 置1 + 调度 ~1000ms 后 `0x080086b4` 清零)。
   - 受其门控的 re-sign 帧(第二分发 `0x0800d4xx` → 改写器簇 `0x08008908+`):
     | 帧 | 改写 | handler |
     |----|------|---------|
     | 0x189 | D0=2 | 0x08008a5c |
     | 0x68c | D3=8 | 0x08008a98 |
     | 0x3a1 | 计数器(D1+1)%15 + D2=0x30 + 表(0x080125e0) | 0x08008aae |
     | 0xa9/0x37a | 返 OTA 协议码 'R'-'W'(0x52-0x57)状态机 | 0x08008908/0x08008b5e |
   - ✅ 已在 `control.c` 忠实实现 0x189/0x68c/0x3a1 + 共享标志。

2. **整车动作注入** (187/167, 待续 步③):
   - `187 handler 0x0800ca48 → 查表 0x0800d668(动作码→内部号) → 0x0800c1b2 → 执行 0x0800bffe`
     (含 4 字节鉴权 key `0x62426141`)。逐动作→确切 CAN 帧 在此链路内, 入口已定位。
   - `167 执行器 0x0800e474`。
   - ✅ 动作码枚举表 `actions.c`(~70 码)已建; 帧字节级待逐项反 0x0800bffe。

3. **解析区监控帧 re-sign** (路径1, 待续 步④): `0x2b6/0x293/0x333/...` 等 12 帧解析时改写回注。

## re-sign 通用约定 (已校正)
- 计数器: **D6 低 nibble +1**(部分帧用 D1 或 (x+1)%15); 校验: **加法校验**(非 CRC8); 见 `VERIFICATION.md`。
