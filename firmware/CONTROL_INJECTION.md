# 控制注入 (re-sign) — 逆向发现记录

> 设备核心控车功能: 拦截特斯拉 CAN 帧 → 改写指定位 → 原 ID 回注。对照 9.bin。

## 两条控制注入路径 + AP 注入
1. **AP/免打扰 注入** (步① 已厘清):
   - 激活标志 `0x28003fb8`(共享)由 **免打扰调度器出口** `0x0800236e` 脉冲置位
     (pulse fn `0x080088ae`: 置1 + 调度 ~1000ms 后 `0x080086b4` 清零)。
   - 受其门控的 re-sign 帧(第二分发 `0x0800d4xx` → 改写器簇 `0x08008908+`):
     | 帧 | 改写 | handler |
     |----|------|---------|
     | 0x189 | D0=2 | 0x08008a5c |
     | 0x68c | D3=8 | 0x08008a98 |
     | 0x3a1 | 计数器(D1+1)%15 + D2=0x30 + 表(0x080125e0) | 0x08008aae |
     | 0xa9/0x37a | 返 OTA 协议码 'R'-'W'(0x52-0x57)状态机 | 0x08008908/0x08008b5e |
   - ✅ 已在 `control.c` 忠实实现 0x189/0x68c/0x3a1 + 共享标志。

2. **整车动作注入** (187/167) — **三层流水线**(步②/步③ 已厘清到合成层):

   **L1 分发**: `187 0x0800ca48 → 查表 0x0800d668(动作码→内部号) → 0x0800c1b2 → 0x0800bffe`(入队/发射框架 `0x0800bbe4`)
   → 动作跳转表 **`0x080127bc`**(`(code-1)*4` 索引, 252 项, `handler=表基+表项; jr`) → ~86 个子系统执行器。
   已**全量提取** → `src/action_map.inc`(252 动作码 → 执行器 + 命令)。

   **L2 命令缓冲**(★关键发现): 执行器**不直接构帧**, 而把"期望动作"字节/字写入 RAM 命令缓冲 **`0x28003f30–0x28003fbc`**
   (每子系统一个槽); 由 L3 周期构建器消费。全量交叉引用槽位图(节选):
   | 槽 | 子系统 | 执行器(写) | 合成器(读→建帧) |
   |----|--------|-----------|------------------|
   | `0x28003f38`(字) | 挡位/拨杆 | `0x08005114` | `0x08004f74` → **re-sign CAN `0x229`** |
   | `0x28003f80` | 门/行李厢/窗 | `0x08005f38` | `0x080058a8`(19 子命令→各 VCSEC 帧, originate) |
   | `0x28003f90..fa7` 块 | 锁/后视镜/窗/前后备厢… | 锁=`0x080078b0`(1锁/2解锁) | `0x0800725a` → **re-sign CAN `0x273`** |
   | `0x28003fb8` | AP/免打扰标志(共享) | 免打扰调度器脉冲 | `0x08008908+` 改写器簇(见路径1) |
   | `0x28003f5c..f7f` | 座椅/喇叭/灯/双闪/驻车… | 各执行器 | 各周期构建器(入口见交叉引用) |

   **L3 合成 / re-sign**: 周期构建器读槽 → 构建/改写对应特斯拉帧(D1/D6 低 nibble 计数器 + 各帧专属表的加法校验)
   → 发送 **`0x0800aaa0`** → `CAN_Transmit(0x40006400=CAN1)`, **原 ID 回注**。

   **★ 端到端实证(挡位)**: 动作 100/101/102(D/R/P) → 执行器 `0x08005114` 写命令字
   {P=`0x1000501`,R=`0x1000201`,D=`0x1000401`}@`0x28003f38` → 构建器 `0x08004f74`(读槽@f38 激活 / @f3a 挡值)
   → 改写**拦截到的 CAN `0x229` SCCM_rightStalk(右拨杆)**(分发臂 `0x0800ce6a`→handler `0x0800d156`→构建器):
   D1 清 bit6:4 + 计数器低 nibble、D2[1:0]=01(有效)、
   D0=校验(表 `0x08012140`) → `j 0x0800aaa0`/CAN1。位布局与 PERID `0x229` 读位(`D1 / D2&~0x70&~3 / D1&0xf`)
   **完全吻合** → **挂挡 = 模拟右拨杆推拨**(与 Model 3/Y 物理换挡一致)。

   **★ 端到端实证(锁/车身)**: 解锁=62/锁=61 → 执行器 `0x080078b0` 写槽@f91(1锁/2解锁) → 车身聚合构建器 `0x0800725a`
   (s2=命令块基址 `0x28003f90`)把 @f90..@fa7 整块命令映射进**拦截到的 CAN `0x273`(UI_vehicleControl)**帧:
   锁→data[2] bits[3:1]、解锁另触发蜂鸣(arg 1000)、后视镜/窗/前后备厢→data[0/3/5/6/7] 各位 → re-sign 回注 0x273。
   触发点 = CAN 分发臂 `0x0800cef6`(`ID==0x273`)→ `0x0800725a`,即**拦截改写**而非凭空发帧。

   **两种触发风格**(分发表实证): (a) **re-sign 拦截改写**——经 CAN 分发表 `0x0800cdfe`(67 臂)触发, 设备等真实帧
   路过再改位回注。控制子集(分发臂→handler→构建器)实证如下表; (b) **动作路径直发**——门/行李厢/窗等**不在分发表**
   (执行器→合成器 `0x080058a8` 从动作路径触发), 推测 originate VCSEC 帧, 确切 ID 待定向反汇编。

   | 控制帧 | 含义 | 分发臂 | handler → 构建器 |
   |--------|------|--------|------------------|
   | `0x229` | 右拨杆(挡位/SWC) | `0x0800ce6a` | `0x0800d156` → `0x08004f74` |
   | `0x273` | UI_vehicleControl(锁/车身聚合) | `0x0800cef6` | `0x0800cefe` → `0x0800725a` |
   | `0x189` | AP D0=2 | `0x0800d49a` | `0x0800d4ba` → `0x08008a5c` |
   | `0x68c` | AP D3=8 | `0x0800d4a2` | `0x0800d4c6` → `0x08008a98` |
   | `0x3a1` | AP 计数器+D2=0x30 | `0x0800d4da` | `0x0800d4fa` → `0x08008aae` |
   | `0x0a9`/`0x37a` | OTA 协议码状态机 | `0x0800d45a`/`0x0800d4ea` | `0x08008908`/`0x08008b5e` |
   | `0x370` | EPAS 无感转向(免打扰) | `0x0800cfb4` | `0x0800d368`(见 ModeMDR 文档) |
   **门**: 执行器 `0x08005f38` 写槽@f80(开门若上锁先调锁 a0=2 解锁) → 合成器 `0x080058a8`(命令 1..19 分发到各子建帧)。

   - ✅ L1 完整映射(`action_map_lookup`) + L2 槽位图 + L3 挡位帧 **已实证**; 其余子系统 L3 确切帧字节
     按上表"合成器"入口可**增量补完**(标 TODO)。逐一穷举 ~40 构建器边际递减, 故先固化模型 + 样例 + 锚点。

3. **解析区监控帧 re-sign** (路径1, 待续 步④): `0x2b6/0x293/0x333/...` 等 12 帧解析时改写回注。

## re-sign 通用约定 (已校正)
- 计数器: **D6 低 nibble +1**(部分帧用 D1 或 (x+1)%15); 校验: **加法校验**(非 CRC8); 见 `VERIFICATION.md`。
