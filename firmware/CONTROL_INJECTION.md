# 控制注入 (re-sign) — 逆向发现记录

> 设备核心控车功能: 拦截特斯拉 CAN 帧 → 改写指定位 → 原 ID 回注。对照 9.bin。

## 两条控制注入路径 + AP 注入
1. **AP/免打扰 注入** (步① 已厘清):
   - 激活标志 `0x28003fb8`(共享)由 **免打扰调度器出口** `0x0800236e` 脉冲置位
     (pulse fn `0x080088ae`: 置1 + 调度 ~1000ms 后 `0x080086b4` 清零)。
   - 受其门控的 re-sign 帧(第二分发 `0x0800d4xx` → 改写器簇 `0x08008908+`):
     | 帧 | 改写 | handler |
     |----|------|---------|
     | 0x189 | D0=2 | 0x08008a5c |
     | 0x68c | D3=8 | 0x08008a98 |
     | 0x3a1 | 计数器(D1+1)%15 + D2=0x30 + 表(0x080125e0) | 0x08008aae |
     | 0xa9/0x37a | 返 OTA 协议码 'R'-'W'(0x52-0x57)状态机 | 0x08008908/0x08008b5e |
   - ✅ 已在 `control.c` 忠实实现 0x189/0x68c/0x3a1 + 共享标志。

2. **整车动作注入** (187/167) — 步② 厘清为 **异步队列+合成**:
   - `187 0x0800ca48 → 查表 0x0800d668(动作码→内部号) → 0x0800c1b2 → 0x0800bffe`。
   - `0x0800bffe` 把动作码 **入队**(`sb a2,0xf(sp); jal 0x0800bbe4` 发射框架), **不在此处直接构帧**;
     由独立**异步任务**取队列 → 合成 CAN 帧 → re-sign → 注入(即 base 文档所述"异步合成 CAN")。
   - 故"逐动作→确切帧字节"在**异步合成阶段**(再深一层); 队列/发射入口已定位(`0x0800bbe4`)。
   - ★ **步②突破**: 找到 187 动作跳转表 **`0x080127bc`**(`(code-1)*4` 索引, 252 项,
     `handler=表基+表项; jr`)。已**全量提取** → `src/action_map.inc`(252 动作码→子系统执行器+命令)。
   - 主要子系统执行器: 门 `0x8005f38` / 挡位 `0x8005114` / 锁 `0x80078b0` / AP `0x800f40c`(38码) /
     座椅 `0x80100f6` / 座椅记忆 `0x80102fa` / 喇叭 `0x80070ae` / 双闪 `0x8007ea0` / 驻车 `0x8005700` 等 86 个。
   - 例: 挂P=102→gear cmd1 · 解锁=62→lock cmd2 · 全门关=133→door cmd13 · AP关=18→AP cmd0。
   - ✅ `action_map_lookup(code)` 暴露完整映射; 各执行器→确切 CAN 帧字节为最后一层(子系统函数, 入口已全给)。

3. **解析区监控帧 re-sign** (路径1, 待续 步④): `0x2b6/0x293/0x333/...` 等 12 帧解析时改写回注。

## re-sign 通用约定 (已校正)
- 计数器: **D6 低 nibble +1**(部分帧用 D1 或 (x+1)%15); 校验: **加法校验**(非 CRC8); 见 `VERIFICATION.md`。
